# Pulse, working notes

Personal Whoop style dashboard for Apple Watch data. SwiftUI iPhone app,
iOS 17+, no dependencies. Phase 2 (planned): workout planner plus exercise
logging from the N=1 project (`~/Documents/Personal Projects/n-equals-one`).

## Build / run

- Simulator device "Pulse-iPhone" (iPhone 17 Pro, iOS 26.3) already exists.
- Build: `xcodebuild -project Pulse.xcodeproj -scheme Pulse -destination
  'platform=iOS Simulator,name=Pulse-iPhone' -derivedDataPath build
  CODE_SIGNING_ALLOWED=NO build`
- Install/launch via `xcrun simctl install/launch … com.wchristie.pulse`.
- Launch args: `-tab 0..3` selects a tab, `-showLatestWorkout` pushes the
  newest workout detail, `-day -N` selects a past day. Use these for CLI
  screenshots.
- Scoring harness: copy `tools/score_check.swift` to `main.swift` in a scratch
  dir, compile with the Models and Scoring files (see git log for the command).

## Architecture

- `Models/`: `HealthDataset` (days plus workouts), Codable, ISO8601 dates.
  Same schema for live HealthKit and the bundled demo JSON.
- `Data/HealthKitSource.swift`: async HK queries into a `HealthDataset`.
- `Data/DemoDataSource.swift`: bundled `Resources/DemoData.json`, date shifted
  so the last day equals today. Generated by `tools/export_to_demo.py` from an
  Apple Health export.xml (the current one is n=1 fixture data, NOT real).
- `Scoring/ScoringEngine.swift`: pure functions. Recovery (z scores vs rolling
  baselines), strain (HR zone load, 0 to 21 log scale), sleep
  (need = base + strain + debt). Calibration constants live here.
- `AppModel` (@Observable, MainActor): tries HealthKit, falls back to demo
  when there are fewer than 7 usable days or auth fails.
- `Views/Theme.swift`: design tokens. Follow the design law: warm paper
  canvas, ink text, hairlines, radius 8 or less, weight 600 only for data
  values, muted green/amber/red ONLY where a metric is a status (recovery,
  sleep performance dots). No gradients, no badges, no tinted progress bars.

## Gotchas

- project.pbxproj is hand written using fileSystemSynchronizedGroups, so new
  files under `Pulse/` are picked up automatically with no pbxproj edits.
- Simulator has no Health data and unsigned builds have no HK entitlement, so
  the app always shows demo data there; that's expected.
- SourceKit single file diagnostics show phantom "cannot find type" errors;
  trust xcodebuild.
